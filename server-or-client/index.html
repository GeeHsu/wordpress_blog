如何判斷後端工程師用得是 SERVER SIDE RENDER 還是 CLIENT SIDE RENDER

2021 年 6 月 8 日 gee.hsu

Node.js 特訓班 – server、client 差異

Server Side render、Client Side render 差異

  • Server Side render：由後端渲染資料
  • Client Side render：由前端渲染資料
    • 跟它為伍，JavaScript 即時呈現
  • 懶人包判斷方式：案右鍵檢視原始碼

如果案右鍵檢視網頁原始碼看到裡面有這段字的時候，就代表說這段話是由後端直接渲染資料給前端使用的。

用 Node.js 來建立一個環境起來

使用終端機建立 express 框架

  1. cd nodeJS
  2. 輸入 express –view ejs
  3. 安裝npm套件，npm install
  4. 執行應用程式，npm start
index.ejs

template language 後端的樣板語言，方便資料庫傳送資料。

講解 index.js、index.ejs 程式碼
// index.js
var express = require('express');
var router = express.Router();

/* GET home page. */
router.get('/', function(req, res, next) {
  res.render('index', { title: 'Express' });

瀏覽器接收到什麼資訊
  1. 按右鍵、按下檢查
  2. 點選 Network
  3. 重新整理
  4. 發現到裡面有一個文件格式
  5. 點一下之後顯示 Response
    代表伺服器回傳到瀏覽器的格式

// routes/index.js
var express = require('express');
var router = express.Router();

/* GET home page. */
router.get('/', function(req, res, next) {
  // 先跟資料庫要資料
  res.render('index', { title: '六角學院' });
});

/* GET search page. */
router.get('/search', function(req, res, next) {
  // 先跟資料庫要資料
  res.render('search', { title: '這裡是搜尋頁面' });
});

/* GET getMore page. */
router.get('/getMore', function(req, res, next) {

  // 會有一段程式事先跟資料庫撈資料

  // 先跟資料庫要資料
  res.send(
    [
      {
        "name": "洧杰",
        "content": "今天超帥"
      },
      {
        "name": "Mary",
        "content": "今天超美"
      },
      {
        "name": "乾太",
        "content": "有 airpods pro" 
      }
    ]
  )
});

module.exports = router;

// views/index.ejs
<!DOCTYPE html>
<html>
  <head>
    <title><%= title %></title>
    <link rel='stylesheet' href='/stylesheets/style.css' />
  </head>
  <body>
    <h1><%= title %></h1>
    <p>Welcome to <%= title %></p>
    <input type="button" name="" value="看更多" class="btn">
    <ul class="list">
      <li>王志誠：他今天有刷牙</li>
      <li>Ray：今天沒有洗臉</li>
    </ul>
    <script>
      document.querySelector('.btn').addEventListener('click', function(){
        var data = null;
        
        var xhr = new XMLHttpRequest();
        xhr.withCredentials = true;

        xhr.addEventListener("readystatechange",
        function() {
          if(this.readyState === 4) {
            let data = JSON.parse(this.responseText);
            let str = '';
            for(let i = 0; data.length > i; i++){
              str += '<li>'+data[i].name+':'+ data[i].content +'</li>'
            }
            document.querySelector('.list').innerHTML += str;
          }
        });

        xhr.open("GET", "http://127.0.0.1:3000/getMore");
        xhr.setRequestHeader("postman-token", "f7bfafd6-0529-1e44-1f4b-4775f9958e55");

        xhr.send(data);
      })
    </script>
  </body>
</html>

上面這個程式碼範例有 Server Side Render、Client(客戶端) Side Render。
AJAX 就算是 Client Side Render 的意思，可是 Render 要提到的一件事是它是顯示畫面給你看。

前兩筆是 Server Side Render，後三筆是 Client Side Render。

資源
  • postman

透過 JS 的方式去跟伺服器要資料。要完資料它在背景回傳資料給我，在 JS 獲得這些值以後再即時顯示、即時渲染這些資料。

Server Side Render 後端工程師合作方式

  1. 前端做好 HTML 頁面
  2. 提交給後端變成動態語言

Client Side Render 後端合作方式

  1. 後端提供給前端 API 列表
  2. 前端依照 API 列表設計網頁介面

抽獎

範例網站練習

  • JS 核心篇 – Server Side Render
  • W3Hexschool 部落格 – Server Side Render
  • 帳號註冊登入 – Client Side Render
  • The F2E 第二屆 – Server Side Render
  • layorder – Client Side Render
    • Vue cli 版型很多、元件很多，
      所有東西都用 Client Side Render 去做
  • 始壯哥健身中心 – Client Side Render
  • 挖栽
  • Facebook – 50% Server Side Render、50% Client Side Render
  • Pinkoi
  • Plurk – 50% Server Side Render、50% Client Side Render
  • Pinterest – 50% Server Side Render、50% Client Side Render
  • 蝦皮購物 – 50% Server Side Render、50% Client Side Render
  • IT邦幫忙 – Server Side Render
  • pixiv – 50% Server Side Render、50% Client Side Render

單純寫在 HTML 上，就不是後端，是靜態網頁(沒有接資料庫)。
所以 HTML 不算 Server Side Render 也不算 Client Side Render。

Q&A – 台灣市場上 Server Side Render、Client Side Render 這兩個比重是多少？
  1. Server 80%、Client 20%
  2. Server 50%、Client 50%
  3. Server 20%、Client 80%

老師認為是1.5，大概是正負10%，未來 Client Side Render 趨勢會變多
